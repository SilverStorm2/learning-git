Plan nauki Git

Krótka ściąga do startu z kontrolą wersji: podstawowe komendy, kluczowe koncepcje i sprawdzone źródła, by szybko wejść w rytm commitów.

Nauka Gita zaczyna się od zrozumienia, że commit to zapisany stan pracy i punkt odniesienia. Praktyka na małych repozytoriach uczy świadomego dodawania zmian oraz bezpiecznego cofania kroków bez stresu. Regularne korzystanie z gałęzi buduje nawyk izolowania funkcji i utrzymywania czytelnej historii projektu.

Celpewne operacje lokalne i na zdalnym repo
Czas3‑5 sesji po 30‑45 min
Mindsetmałe commity, częsty push

Podstawowe komendy

  • git status – sprawdź co zmienione
  • git add . – dodaj zmiany do commita
  • git commit -m "opis" – zapisz punkt w historii
  • git log --oneline --graph – szybki podgląd historii
  • git branch / git checkout -b feature – praca na gałęziach
  • git push / git pull – synchronizacja ze zdalnym

Na co uważać

  • Commity rób małe i opisowe – łatwiej cofnąć lub zreviewować.
  • Przed pushem: git status i szybki git diff.
  • Nie nadpisuj historii wspólnej (git push --force) bez uzgodnienia.
  • Unikaj pracy na master/main; używaj gałęzi tematycznych.

Ćwiczenia na start

  • Stwórz repo, zrób 3 commity (np. README, plik JS, poprawka).
  • Załóż gałąź feature/notes, dodaj plik i zintegruj z main.
  • Rozwiąż prosty konflikt: zmień tę samą linię w dwóch gałęziach i zmerguj.

Głębiej

  • Staging vs working tree: co trafia do commita.
  • Rebase vs merge: kiedy porządkować historię, a kiedy zachować gałęzie.
  • Stash: git stash i git stash pop przy szybkiej zmianie kontekstu.
